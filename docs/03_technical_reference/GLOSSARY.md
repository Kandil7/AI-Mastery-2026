# AI-Mastery-2026 Glossary

Comprehensive glossary of terms used in AI-Mastery-2026.

---

## A

**Activation Function**
A function that determines the output of a neural network node. Common examples include ReLU, sigmoid, and tanh.

**Adapter**
A fine-tuning method that adds small trainable layers to a pre-trained model while keeping most parameters frozen.

**Adam**
Adaptive Moment Estimation - an optimization algorithm that combines momentum and RMSprop.

**Attention**
A mechanism that allows models to focus on relevant parts of input when processing sequences.

**Autograd**
Automatic differentiation - the system for computing gradients automatically.

---

## B

**Backpropagation**
The algorithm for computing gradients in neural networks by propagating errors backward.

**Batch Size**
The number of samples processed before updating model parameters.

**BERT**
Bidirectional Encoder Representations from Transformers - a transformer encoder model.

**Bias**
An additive parameter in neural networks that allows shifting the activation function.

**BM25**
Best Matching 25 - a probabilistic retrieval function used in information retrieval.

---

## C

**Cache**
A high-speed data storage layer that stores frequently accessed data.

**Chain**
A sequence of operations or components connected together.

**Chunking**
The process of splitting documents into smaller pieces for processing.

**CNN**
Convolutional Neural Network - a neural network architecture for processing grid-like data.

**Config**
Short for configuration - settings that control program behavior.

**Cosine Similarity**
A measure of similarity between two vectors based on the cosine of the angle between them.

---

## D

**DataLoader**
A utility for loading data in batches with optional shuffling and parallel loading.

**Deep Learning**
A subset of machine learning using multi-layer neural networks.

**Document**
A unit of text or content in RAG systems.

**Dropout**
A regularization technique that randomly sets activations to zero during training.

---

## E

**Embedding**
A dense vector representation of discrete data (words, tokens, etc.).

**Encoder**
A model component that transforms input into a latent representation.

**Epoch**
One complete pass through the entire training dataset.

**Evaluation**
The process of assessing model performance on test data.

---

## F

**FAISS**
Facebook AI Similarity Search - a library for efficient similarity search.

**Fine-tuning**
The process of adapting a pre-trained model to a specific task.

**Flash Attention**
An optimized attention implementation that reduces memory usage.

**Forward Pass**
The computation of outputs from inputs through a neural network.

---

## G

**GPT**
Generative Pre-trained Transformer - a family of transformer decoder models.

**Gradient**
The vector of partial derivatives indicating the direction of steepest ascent.

**Gradient Descent**
An optimization algorithm that minimizes loss by moving in the direction of negative gradient.

**Graph Neural Network (GNN)**
A neural network that operates on graph-structured data.

---

## H

**Hidden Dimension**
The size of the latent representations in a neural network.

**HNSW**
Hierarchical Navigable Small World - an algorithm for approximate nearest neighbor search.

**HyDE**
Hypothetical Document Embeddings - a retrieval technique using generated documents.

---

## I

**Inference**
The process of using a trained model to make predictions.

**Integration**
Combining multiple components or systems to work together.

---

## K

**K-Nearest Neighbors (KNN)**
An algorithm that classifies based on the majority class among k nearest neighbors.

**Kernel**
A function that computes inner products in a high-dimensional space (SVM).

---

## L

**Layer**
A computational unit in a neural network that transforms its input.

**Learning Rate**
A hyperparameter that controls the step size in optimization.

**LoRA**
Low-Rank Adaptation - a parameter-efficient fine-tuning method.

**Loss Function**
A function that measures the difference between predictions and targets.

**LSTM**
Long Short-Term Memory - a type of recurrent neural network.

---

## M

**MCMC**
Markov Chain Monte Carlo - methods for sampling from probability distributions.

**Memory**
Storage for conversation history or context in agents.

**Mini-batch**
A small subset of the training data used for one optimization step.

**Multi-Head Attention**
Attention mechanism with multiple parallel attention heads.

---

## N

**Neural Network**
A computational model inspired by biological neural networks.

**NUTS**
No-U-Turn Sampler - an extension of Hamiltonian Monte Carlo.

---

## O

**Optimizer**
An algorithm that updates model parameters to minimize loss.

**Overfitting**
When a model learns training data too well and fails to generalize.

---

## P

**Parameter**
A variable in a model that is learned from data.

**Pipeline**
A sequence of processing steps connected together.

**Pooling**
An operation that reduces the spatial dimensions of feature maps.

**Pre-training**
Training a model on a large dataset before fine-tuning.

**Protocol**
A type definition that specifies required methods (Python typing).

---

## Q

**Query**
A request for information in retrieval systems.

**Quantization**
Reducing the precision of model weights for efficiency.

---

## R

**RAG**
Retrieval-Augmented Generation - combining retrieval with generation.

**ReAct**
Reasoning and Acting - an agent architecture combining reasoning and action.

**Regularization**
Techniques to prevent overfitting by adding constraints.

**ResNet**
Residual Network - a CNN architecture with skip connections.

**Retrieval**
The process of finding relevant documents for a query.

**Reranking**
Reordering retrieved results based on additional criteria.

**RoPE**
Rotary Positional Embeddings - a method for encoding position.

---

## S

**Semantic Search**
Search based on meaning rather than keyword matching.

**Similarity**
A measure of how alike two items are.

**Softmax**
A function that converts logits to probabilities.

**Stochastic**
Involving randomness or probability.

**SVM**
Support Vector Machine - a classification algorithm.

---

## T

**Temperature**
A parameter controlling randomness in text generation.

**Token**
A unit of text (word, subword, or character) in NLP.

**Tool**
A function or capability that an agent can use.

**Transformer**
A neural network architecture based on self-attention.

**Training**
The process of learning model parameters from data.

---

## V

**Validation**
Evaluating model performance during training for hyperparameter tuning.

**Vector Store**
A database optimized for storing and searching vector embeddings.

**Vocabulary**
The set of unique tokens a model can process.

---

## W

**Weight**
A multiplicative parameter in neural networks.

**Word Embedding**
Vector representation of words.
