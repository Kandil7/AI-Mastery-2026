## 🎯 Curriculum Philosophy

### Core Principles

1. **Math First → Code Second → Libraries Third → Production Always**
   - Derive gradients and proofs on paper before implementing
   - Build from-scratch implementations before using frameworks
   - Transition to production libraries after understanding fundamentals
   - Every component must be production-ready

2. **Competency-Based Progression**
   - Advance upon demonstrated mastery, not time spent
   - Multiple assessment modalities (quizzes, projects, peer review)
   - Clear rubrics for every evaluation
   - Remediation paths for struggling learners

3. **Industry-Aligned Skills**
   - Map every module to job competencies
   - Include industry-standard tools (LangChain, MLflow, Vector DBs)
   - Real-world projects with business impact
   - Portfolio-building deliverables

4. **AI-Specific UX Patterns**
   - Streaming responses with typing indicators
   - Citations for all factual claims
   - Confidence display for uncertain answers
   - Human handoff options

### Bloom's Taxonomy Integration

Every learning objective uses specific Bloom's taxonomy verbs:

| Level | Verbs Used | Assessment Type |
|-------|-----------|-----------------|
| **Remember** | Define, List, Recall, Identify | Multiple choice, Flashcards |
| **Understand** | Explain, Describe, Summarize | Short answer, Diagrams |
| **Apply** | Implement, Use, Execute, Calculate | Coding exercises, Labs |
| **Analyze** | Compare, Contrast, Debug, Diagnose | Case studies, Code review |
| **Evaluate** | Critique, Justify, Recommend, Assess | Project reviews, Presentations |
| **Create** | Design, Build, Architect, Optimize | Capstone projects, Research |

### Sample Module Specification

#### Module 3.2: Transformer Architecture Deep Dive

**Duration:**
- Theory: 5 hours
- Hands-on: 6 hours
- Assessment: 3 hours
- **Total:** 14 hours

**Learning Objectives:**
By the end of this module, learners will be able to:
1. **Explain** each component of the transformer architecture (Understand)
2. **Implement** positional encoding from scratch (Apply)
3. **Compare** encoder-only, decoder-only, and encoder-decoder architectures (Analyze)
4. **Calculate** attention scores manually (Apply)
5. **Debug** common transformer implementation issues (Analyze)

**Prerequisites:**
- Module 3.1: NLP Fundamentals for LLMs
- Module 2.5: Neural Networks Fundamentals
- Python proficiency

**Content Outline:**
1. Transformer Overview (30 min)
   - Historical context
   - Attention is All You Need paper
   - Impact on NLP

2. Self-Attention Mechanism (60 min)
   - Query, Key, Value concept
   - Scaled dot-product attention
   - Multi-head attention

3. Positional Encoding (45 min)
   - Why position matters
   - Sinusoidal encoding
   - Learned embeddings

4. Feed-Forward Networks (30 min)
   - Position-wise FFN
   - Activation functions

5. Layer Normalization & Residuals (45 min)
   - Pre-norm vs post-norm
   - Residual connections

6. Encoder-Decoder Structure (60 min)
   - Encoder stack
   - Decoder stack
   - Cross-attention

7. Variants & Evolution (45 min)
   - BERT (encoder-only)
   - GPT (decoder-only)
   - T5 (encoder-decoder)

**Hands-on Exercises:**

1. **Exercise 1: Implement Positional Encoding**
   - Duration: 90 minutes
   - Deliverable: `positional_encoding.py` with tests
   - Requirements: Support both sinusoidal and learned encoding

2. **Exercise 2: Build Self-Attention Layer**
   - Duration: 120 minutes
   - Deliverable: `self_attention.py` with tests
   - Requirements: Scaled dot-product, masking support

3. **Exercise 3: Multi-Head Attention**
   - Duration: 90 minutes
   - Deliverable: `multi_head_attention.py`
   - Requirements: Configurable heads, proper concatenation

**Knowledge Check (5 questions):**
1. What is the purpose of scaling in scaled dot-product attention?
2. Why do we need positional encoding in transformers?
3. How many parameters does a multi-head attention layer have with d_model=512 and 8 heads?
4. What is the difference between self-attention and cross-attention?
5. Why are residual connections important in deep transformers?

**Resources:**
- Reading: "Attention is All You Need" (Vaswani et al., 2017)
- Reading: "The Illustrated Transformer" (Jay Alammar)
- Video: Transformer architecture walkthrough
- Code: Reference implementation in src/llm/transformer.py

**Assessment Criteria:**
- Positional encoding produces correct outputs (verified by tests)
- Self-attention handles masking correctly
- Multi-head attention splits and concatenates properly
- Code is well-documented and typed
- All tests pass
