## 🔑 Key Formulas

### Time Estimation

```
Total Hours = Theory Hours + Hands-on Hours + Assessment Hours

Per Module:
- Theory: 30-40% of total
- Hands-on: 50-60% of total
- Assessment: 10-20% of total
```

### Grade Calculation

```
Track Grade = (Quiz Avg × 0.20) + (Project Avg × 0.40) + (Capstone × 0.25) + (Knowledge Checks × 0.10) + (Peer Reviews × 0.05)
```

### Progress Tracking

```
Completion % = (Modules Completed / Total Modules) × 100

On Track = (Hours Spent / Expected Hours) ≤ 1.2
```

## 📊 Bloom's Taxonomy Quick Reference

### Levels and Verbs

| Level | Verbs | Assessment Types |
|-------|-------|------------------|
| **Remember** | Define, List, Recall, Identify | Multiple choice, Flashcards |
| **Understand** | Explain, Describe, Summarize | Short answer, Diagrams |
| **Apply** | Implement, Use, Execute, Calculate | Coding exercises, Labs |
| **Analyze** | Compare, Contrast, Debug, Diagnose | Case studies, Code review |
| **Evaluate** | Critique, Justify, Recommend, Assess | Project reviews, Presentations |
| **Create** | Design, Build, Architect, Optimize | Capstone projects, Research |

## 🎯 Quick Tips for Success

### Time Management
1. **Block dedicated study time** - Same time each day
2. **Use Pomodoro technique** - 25 min focus, 5 min break
3. **Start with theory** - Then immediately apply
4. **Review before bed** - Improves retention

### Learning Strategies
1. **Code along** - Don't just read examples
2. **Build projects** - Learning by doing
3. **Teach others** - Best way to learn
4. **Join study groups** - Collaborative learning

### Assessment Tips
1. **Review learning objectives** - Before each quiz
2. **Practice coding** - Daily, even 30 minutes
3. **Start projects early** - Allow time for iteration
4. **Seek feedback** - From peers and mentors

### Wellness
1. **Take breaks** - Prevent burnout
2. **Sleep well** - Critical for learning
3. **Exercise** - Improves cognitive function
4. **Stay hydrated** - Often overlooked
