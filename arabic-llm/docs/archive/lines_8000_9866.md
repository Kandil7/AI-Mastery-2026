```
balygh_score = 0.4 * fiqh_hadith + 0.3 * lang + 0.3 * scraping

Where:
- fiqh_hadith = 0.6 * fiqh_f1 + 0.4 * hadith_f1
- lang = 0.7 * nahw_score + 0.3 * balagha_score
- scraping = 0.4 * json_acc + 0.6 * field_f1
```

- Fiqh QA (F1 score)
- Hadith takhreej (F1 score)
- Arabic grammar parsing (ROUGE-L)
- Rhetoric analysis (ROUGE-L)

## 🚀 Usage Guide

### Step 1: Generate SFT Dataset from Books

```bash
cd K:\learning\technical\ai-ml\AI-Mastery-2026\arabic-llm

# Generate 5,000 examples from 200 books
python scripts/build_balygh_sft_dataset.py \
  --target-examples 5000 \
  --max-books 200

# Output: datasets/jsonl/balygh_sft_from_books.jsonl
```

**What it does:**
1. Loads book metadata from `datasets/metadata/books.json`
2. Reads extracted books from `datasets/extracted_books/`
3. Splits into paragraphs (150-1200 chars)
4. Categorizes books (fiqh, hadith, tafsir, language, literature)
5. Generates SFT examples using role-specific templates
6. Outputs JSONL with placeholder outputs (output_stub)

### Step 2: Refine Outputs with LLM

```bash
# Refine using DeepSeek (cheap, fast)
export DEEPSEEK_API_KEY="sk-..."

python scripts/refine_balygh_sft_with_llm.py \
  --max-examples 1000 \
  --provider deepseek

# Output: datasets/jsonl/balygh_sft_refined.jsonl
```

**Supported Providers:**
- DeepSeek (recommended for cost/speed)
- Together AI (Qwen2.5-72B)
- Groq (Llama-3.1-70B)
- OpenAI (GPT-4o-mini)

**What it does:**
1. Reads raw SFT examples with `output_stub`
2. Builds role-specific prompts with strict rules
3. Calls LLM to generate high-quality outputs
4. Cleans and validates generated text
5. Saves refined examples with quality scores

### Step 3: Train Model

```bash
# Standard training
python scripts/03_train_model.py \
  --config configs/training_config.yaml \
  --dataset datasets/jsonl/balygh_sft_refined.jsonl \
  --output-dir models/balygh-v2

# Or with Unsloth (2x faster)
python scripts/train.py \
  --model-name Qwen/Qwen2.5-7B-Instruct \
  --dataset datasets/jsonl/balygh_sft_refined.jsonl \
  --lora-r 64 \
  --epochs 3
```

### Step 4: Evaluate with Balygh Score

```bash
# Set model directory
export BALYGH_MODEL_DIR="models/balygh-v2"
export BALYGH_EVAL_DIR="datasets/evaluation"

# Run evaluation
python scripts/prepare.py

# Output includes:
# - Detailed JSON metrics
# - Single line: balygh_score=0.XXXX (for autoresearch)
```

## 🔧 AutoResearch Integration

### program.md for AutoResearch

Create `autoresearch/program.md`:

```markdown
# AutoResearch Program for Balygh (Arabic Qwen2.5)

## Goal

Optimize fine-tuning of **Qwen2.5-7B-Instruct** on the **Balygh Arabic SFT dataset**
for **Arabic fiqh + linguistics**.

The objective metric is `balygh_score` returned by `prepare.py`:
- `balygh_score` ∈ [0, 1]
- higher is better
- combines: fiqh, hadith, nahw, balagha, scraping

## Files

- `train.py` — defines `train(config)` using QLoRA + Unsloth + Qwen2.5
- `prepare.py` — loads checkpoint and computes `balygh_score`
- `config.json` — current hyperparameters

The agent **MAY** edit: `config.json`, `train.py` (only hyperparams)
The agent **MUST NOT** edit: `prepare.py`, eval data, model weights

## Search Space

The agent should explore combinations of:

- `learning_rate`: [5e-5, 1e-4, 2e-4]
- `lora_r`: [32, 64]
- `lora_alpha`: always `2 * lora_r`
- `num_train_epochs`: [2, 3]
- `max_seq_length`: [2048, 3072, 4096]
- `per_device_train_batch_size`: [2, 4]
- `gradient_accumulation_steps`: adjust to keep effective batch ≈ 16
- `warmup_ratio`: [0.03, 0.05]
- `weight_decay`: [0.0, 0.01]
- `packing`: [true, false]

## Evaluation Budget

- Each `train.py` run must not exceed **90 minutes** on a single GPU
- For each candidate config:
  1. Run `python train.py --config config.json --fast`
  2. Run `python prepare.py` → prints `balygh_score=<value>`
  3. If `balygh_score` improved over best so far by at least `0.01`:
     - keep changes (git commit)
  4. Otherwise:
     - revert changes (git reset)

## Constraints

- Do **not** modify model base (stay on Qwen2.5-7B-Instruct)
- Keep LoRA-only training (no full finetune)
- Do not increase VRAM usage beyond current config
- Do not change eval data or metric definitions
- Prefer **simpler** configs when scores are tied

## Strategy Hints

1. Start from baseline: `lr=2e-4, r=64, seq=4096, epochs=3, packing=true`
2. First, tune `learning_rate` and `lora_r`
3. Then, tune `max_seq_length` if VRAM/time allows
4. Finally, fine-tune regularization (weight_decay, warmup_ratio)

## Stopping Criteria

Stop when:
- No improvement ≥ 0.01 for 10 consecutive experiments, or
- Maximum of 50 experiments is reached
```

## 📚 Key Insights from Lines 8000-9866

### Data-Centric Approach

1. **Start with high-quality nucleus** (15-20 carefully selected books) instead of dumping everything
2. **Use LLM to refine outputs** - raw stubs → high-quality answers
3. **Iterate and expand** gradually after stabilizing performance

### AutoResearch Pattern

1. **Agent modifies** train.py or config.json
2. **Runs training** for short period (90 min)
3. **Evaluates** with prepare.py → balygh_score
4. **Keeps** improvements (git commit) or **reverts** (git reset)
5. **Repeats** until convergence

### Best Practices from Arabic LLM Projects

**From ArabicWeb24, Tahakom, AceGPT, ALLaM:**

1. **Deduplication is critical** - Document + sentence level
2. **Language filtering** - fastText Arabic ID ≥ 0.65
3. **Quality filtering** - Gopher/FineWeb filters
4. **Tokenization audit** - Measure fertility before/after
5. **Dialect policy** - Clear separation MSA vs dialects
6. **Safety layer** - fatwa_assistant_safe role with disclaimers

### Tokenization Recommendations

1. **Use SentencePiece Unigram** for Arabic (better than BPE)
2. **Vocab size 64-80k** for Arabic-only models
3. **Don't change tokenizer** after pre-training without retraining
4. **Audit fertility** - target 1.2-1.5 tokens/word

### Evaluation Best Practices

1. **Multi-component metric** (fiqh + language + scraping)
2. **Role-specific tests** for all 29 roles
3. **OALL benchmarks** for comparison
4. **Custom linguistics tests** (i'rab, balagha, poetry)
