## 🎯 Project Overview

**Balygh (بليغ)** is a specialized Arabic LLM system built from the comprehensive implementation plan in `llm_arabic_plan.md` (11,993 lines).

### Key Capabilities

- **29 specialized roles** across 5 categories
- **76 linguistic and Islamic skills**
- **100,000 training examples** target
- **Multi-domain expertise**: Fiqh, Hadith, Tafsir, Arabic Grammar, Rhetoric, RAG
- **Safety-first approach** for Islamic sciences
- **Production-ready** with QLoRA fine-tuning

### Base Configuration

- **Base Model**: Qwen2.5-7B-Instruct
- **Training Method**: QLoRA (r=64, alpha=128)
- **Sequence Length**: 4096 tokens
- **Training Time**: ~12 hours on 24GB GPU
- **Target Dataset**: 100K examples

**Status**: ✅ Complete

**7-Stage Cleaning Pipeline**:
```python
Stage 1: Encoding cleanup (BOM, mojibake)
Stage 2: Unicode NFC normalization
Stage 3: Arabic normalization (ألف، تاء مربوطة)
Stage 4: Control characters removal
Stage 5: Whitespace normalization
Stage 6: OCR error correction
Stage 7: Punctuation normalization
```

**3-Level Deduplication**:
```python
Level 1: Exact dedup (SHA-256 hash)
Level 2: Near-duplicate (MinHash LSH, threshold=0.8)
Level 3: Sentence-level dedup (3-sentence spans)
```

balygh_score = 0.4 * fiqh_hadith + 0.3 * lang + 0.3 * scraping

## 🏋️ Training Pipeline

### Complete Training Flow

```bash
# Step 1: Prepare dataset
python scripts/build_balygh_sft_dataset.py --target-examples 50000

# Step 2: Refine with LLM
python scripts/refine_balygh_sft_with_llm.py --max-examples 50000

# Step 3: Merge datasets
python src/merge_datasets.py

# Step 4: Train model
python scripts/03_train_model.py \
  --config configs/training_config.yaml \
  --dataset datasets/jsonl/balygh_sft_v1.jsonl \
  --output-dir models/balygh-v2

# Step 5: Evaluate
export BALYGH_MODEL_DIR="models/balygh-v2"
python scripts/prepare.py
```

### Gradio Demo

```python
import gradio as gr
from transformers import AutoTokenizer, AutoModelForCausalLM

model_id = "YourUser/Balygh-7B-v1"
tokenizer = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForCausalLM.from_pretrained(model_id, device_map="auto")

def chat(prompt, max_new_tokens=256, temperature=0.3):
    inputs = tokenizer(prompt, return_tensors="pt").to(model.device)
    outputs = model.generate(**inputs, max_new_tokens=max_new_tokens, temperature=temperature)
    return tokenizer.decode(outputs, skip_special_tokens=True)

iface = gr.Interface(fn=chat, inputs=gr.Textbox(), outputs=gr.Textbox())
iface.launch()
```

## ✅ Best Practices

### From Implementation Plan (Lines 1-11993)

1. **Data-Centric Approach**
   - Start with high-quality nucleus (15-20 books)
   - Use LLM to refine outputs
   - Iterate and expand gradually

2. **Deduplication is Critical**
   - Document-level MinHash (threshold=0.8)
   - Sentence-level for repeated chains/intros

3. **Safety-First for Islamic Sciences**
   - Separate `fatwa_assistant_safe` role
   - Mandatory disclaimers
   - DPO for preference tuning

4. **Tokenization Audit**
   - Measure fertility before/after
   - Target 1.2-1.5 tokens/word for Arabic

5. **Multi-Turn Training**
   - Convert single-turn to multi-turn
   - Include clarifying questions
   - Step-by-step reasoning

6. **Evaluation Battery**
   - OALL benchmarks
   - Custom linguistics tests
   - Role-specific evaluations

7. **AutoResearch Integration**
   - Automated hyperparameter search
   - Balygh score as objective metric
   - Git-based experiment tracking
